Installation

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory moved/ to your server into CMSimple_XH's plugins directory.
  4. Set write permissions to the subdirectories config/, languages/ and the plugin's content folder.
  5. Switch to "Moved" in the back-end to check if all requirements are fulfilled.
back to top

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select "Moved" from "Plugins".

You can change the default settings of Moved_XH in "Config". Hints for the options will be displayed when hovering over the help icon with your mouse.

Localization is done in "Language". You can translate the character strings to your own language, or customize them according to your needs.

Limitations

  • Moved_XH uses the custom_404() hook, so it doesn't work if the hook is alread defined. In this case you'll get a blank browser window when you browse to your website. Either uninstall Moved_XH or remove the existing custom_404() hook.
  • Moved_XH doesn't work when you're logged to the CMSimple_XH installation, as the custom_404() hook funktion is not called by CMSimple_XH in this case. If you want to test the moved pages without login out, you have to use a second browser.
